# Kermesse Bois Luzy

Application React/Vite de réservation de créneaux pour la kermesse de l'école maternelle de Bois Luzy.

## Lancer le projet

```bash
npm install
npm run dev
```

## Déploiement Vercel

Le dépôt contient `vercel.json`. Dans Vercel, importer le dépôt GitHub puis renseigner ces variables dans `Project Settings > Environment Variables` :

```bash
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
VITE_ADMIN_EMAIL=
```

Paramètres attendus :

- Build command : `npm run build`
- Output directory : `dist`
- Framework : `Vite`

## Variables d'environnement

Créer un fichier `.env` à partir de `.env.example`.

```bash
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
VITE_ADMIN_EMAIL=
```

Le mot de passe admin ne doit pas être stocké dans une variable `VITE_*`, car ces variables sont incluses dans le JavaScript envoyé au navigateur. L'administration utilise Supabase Auth avec un compte email/mot de passe, et la session est mémorisée automatiquement par Supabase.

Le mot de passe admin est celui du compte créé dans Supabase :

1. Aller dans Supabase > Authentication > Users.
2. Créer un utilisateur avec l'email renseigné dans `VITE_ADMIN_EMAIL`.
3. Choisir le mot de passe dans Supabase.
4. Utiliser cet email et ce mot de passe dans l'espace admin du site.

Si l'application affiche `Failed to fetch`, vérifier que `VITE_SUPABASE_URL` ressemble à `https://xxxxx.supabase.co` et que `VITE_SUPABASE_ANON_KEY` est bien la clé `anon public` complète du projet Supabase.

Si Supabase affiche une erreur de relation entre `stands` et `slots`, vérifier que la colonne `slots.stand_id` existe bien et référence `stands(id)`. Le code ne dépend plus du cache de relation Supabase pour charger les créneaux, mais cette clé étrangère reste utile pour l'intégrité des données.

## Schéma Supabase

Exécuter ce SQL dans Supabase, puis créer un utilisateur Auth avec l'email `VITE_ADMIN_EMAIL`.

```sql
create table if not exists stands (
  id bigint generated by default as identity primary key,
  name text not null,
  emoji text not null default '🎪',
  created_at timestamptz not null default now()
);

create table if not exists slots (
  id bigint generated by default as identity primary key,
  stand_id bigint not null references stands(id) on delete cascade,
  time text not null,
  created_at timestamptz not null default now()
);

create table if not exists reservations (
  id bigint generated by default as identity primary key,
  slot_id bigint not null unique references slots(id) on delete cascade,
  parent_name text not null,
  phone text not null,
  relationship text not null,
  child_name text not null,
  teacher_name text,
  created_at timestamptz not null default now()
);

create or replace view public_reservation_status as
select slot_id
from reservations;

alter table stands enable row level security;
alter table slots enable row level security;
alter table reservations enable row level security;

grant select on public_reservation_status to anon, authenticated;

create policy "Public read stands"
on stands for select
to anon, authenticated
using (true);

create policy "Public read slots"
on slots for select
to anon, authenticated
using (true);

create policy "Public create reservations"
on reservations for insert
to anon, authenticated
with check (true);
```

### Policies admin

Dans Supabase > Authentication > Users, copier l'`User UID` du compte admin, puis exécuter ce SQL en remplaçant `00000000-0000-0000-0000-000000000000` par cet UID.

Ne pas faire de `select` vers `auth.users` dans les policies : selon le contexte d'exécution, cela peut provoquer `permission denied for table users`.

```sql
drop policy if exists "Admin manage stands" on stands;
drop policy if exists "Admin manage slots" on slots;
drop policy if exists "Admin read reservations" on reservations;
drop policy if exists "Admin manage reservations" on reservations;
drop policy if exists "Admin delete reservations" on reservations;

create policy "Admin manage stands"
on stands for all
to authenticated
using (auth.uid() = '00000000-0000-0000-0000-000000000000')
with check (auth.uid() = '00000000-0000-0000-0000-000000000000');

create policy "Admin manage slots"
on slots for all
to authenticated
using (auth.uid() = '00000000-0000-0000-0000-000000000000')
with check (auth.uid() = '00000000-0000-0000-0000-000000000000');

create policy "Admin read reservations"
on reservations for select
to authenticated
using (auth.uid() = '00000000-0000-0000-0000-000000000000');

create policy "Admin delete reservations"
on reservations for delete
to authenticated
using (auth.uid() = '00000000-0000-0000-0000-000000000000');
```

## Données de départ

Exemple pour retrouver les deux stands de base :

```sql
insert into stands (name, emoji)
values
  ('Pêche aux canards', '🦆'),
  ('Tir à l''arc', '🎯');

insert into slots (stand_id, time)
select id, '10h00 - 10h30' from stands where name = 'Pêche aux canards'
union all
select id, '10h30 - 11h00' from stands where name = 'Pêche aux canards'
union all
select id, '11h00 - 11h30' from stands where name = 'Tir à l''arc'
union all
select id, '11h30 - 12h00' from stands where name = 'Tir à l''arc';
```

## Structure

- `src/components` : composants UI parents, admin et modales.
- `src/hooks` : état applicatif et session admin.
- `src/services` : client Supabase et requêtes.
- `src/utils` : constantes et export Excel.
